## Day 2：数据流拆解（输入怎么变成训练信号）

### 教学目标

理解：**一句话是怎么进入 GPT 并变成 loss 的**

### 核心代码

重点看 `train.py` 中的：

- `get_batch()` 函数（约第 70-85 行）
- 数据文件格式：`train.bin` / `val.bin` 是 uint16 numpy 数组

### 数据流图（自己画一遍）

```
原始文本
  ↓
tokenizer (字符级 or BPE)
  ↓
token IDs (uint16 数组)
  ↓
np.memmap 读取
  ↓
get_batch() 随机切块
  ↓
x = tokens[i : i+block_size]        # 输入
y = tokens[i+1 : i+1+block_size]    # 标签（右移一位）
  ↓
model(x) → logits
  ↓
CrossEntropyLoss(logits, y) → loss
  ↓
loss.backward() → 梯度更新
```

### 必须搞清楚的概念

| 概念 | 解释 |
|------|------|
| `block_size` | 上下文窗口长度（最大能看多远的 token） |
| `x` 和 `y` 的关系 | `y` 是 `x` 右移一位，因为任务是"预测下一个 token" |
| `batch_size` | 一次送多少条独立序列进模型 |
| 为什么用 `np.memmap` | 数据太大（17GB）不能全放内存，按需读取 |

### 动手实验

- [ ] 打印 `x` 和 `y` 的 shape，理解维度含义
- [ ] 把 `batch_size` 从 12 改成 2，观察训练速度变化
- [ ] 把 `block_size` 从 256 改成 64，观察生成质量变化
- [ ] 打印一个 batch 的 token，用 tokenizer 解码回文本看看

### 输出成果

- 一张手绘/电子版"数据流图"
- 能口述：从原始文本到 loss 的完整流程

## Day 3：Transformer 核心 — Attention 机制

### 教学目标

理解 GPT 的"灵魂"：**Self-Attention**

### 核心代码

`model.py` 中的 `CausalSelfAttention` 类（约第 50-90 行）

### 关键概念

#### Q / K / V 是什么

```
Q (Query) = "我在找什么信息"
K (Key)   = "我能提供什么信息"
V (Value) = "我实际的信息内容"

Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

#### 因果 Mask

```
不使用 mask 时：
token 3 可以看到 token 1, 2, 3, 4, 5（违反因果）

使用 causal mask 后：
token 3 只能看到 token 1, 2, 3（正确）
```

mask 是一个上三角矩阵，把未来位置设为 `-inf`，softmax 后变成 0。

#### Multi-Head Attention

```
不是用一个大 attention，而是用多个小 attention（head）并行
每个 head 学不同的"关注模式"
最后 concat 起来
```

### 动手实验

- [ ] 打印 `attn` 的 shape，理解 `(B, nh, T, T)` 的含义
- [ ] 把 `n_head` 从 6 改成 2，观察 loss 变化（表达能力下降）
- [ ] 把 `n_head` 从 6 改成 12，观察参数量变化
- [ ] 去掉 causal mask（注释掉），看模型是否还能正常训练（会泄露未来信息）

### 一句话解释 Attention

> "每个 token 在看之前所有 token 的信息，通过计算相似度来决定关注程度，然后加权融合。"

### 输出成果

- 能用一句话解释 attention
- n_head=2 vs n_head=6 的 loss 对比记录

## Day 4：Transformer Block 全拆

### 教学目标

理解一个完整的 Transformer Block 是怎么组合的

### 核心代码

`model.py` 中的 `Block` 类和 `GPT` 类

### Block 结构图

```
输入 x
  ├───────────────┐
  ↓               │
LayerNorm         │ (residual)
  ↓               │
CausalSelfAttention
  ↓               │
  + ←─────────────┘  (x + attn(ln(x)))
  ├───────────────┐
  ↓               │
LayerNorm         │ (residual)
  ↓               │
MLP (FFN)
  ↓               │
  + ←─────────────┘  (x + mlp(ln(x)))
  ↓
输出
```

### 各组件作用

| 组件 | 作用 | 去掉会怎样 |
|------|------|-----------|
| LayerNorm | 稳定训练，归一化中间值 | 训练不稳定，loss 震荡或爆炸 |
| Residual Connection | 让梯度直接流过，解决深层网络梯度消失 | 深层网络很难训练 |
| MLP (FFN) | 存储知识/记忆，扩展表达能力 | 模型只能做 attention，表达力大降 |
| Dropout | 防止过拟合 | 小数据集上过拟合 |

### GPT 整体架构

```
Token Embedding (wte) + Position Embedding (wpe)
  ↓
Dropout
  ↓
N × Transformer Block
  ↓
Final LayerNorm
  ↓
Linear (lm_head)  ← 与 wte 权重共享（weight tying）
  ↓
Logits (vocab_size)
```

### 动手实验

- [ ] 注释掉 LayerNorm，观察 loss 是否爆炸
- [ ] 把 MLP 隐藏层从 `4 * n_embd` 改成 `2 * n_embd`，观察效果
- [ ] 把 `dropout` 从 0.0 改成 0.2，在小数据集上观察过拟合现象
- [ ] 打印模型参数量：`sum(p.numel() for p in model.parameters())`

### 输出成果

- 一张自己画的 GPT Block 结构图
- 消融实验记录（去掉某个组件后的 loss 对比）

## Day 5：训练机制（真正的"LLM 工程味"）

### 教学目标

理解：**训练为什么能收敛，以及什么会导致训练崩溃**

### 核心代码

`train.py` 中的优化器配置和训练循环

### 关键概念

#### AdamW 优化器

```
不是普通 SGD，而是自适应学习率
每个参数有自己的"动量"和"速度"
weight decay 不加在梯度上，而是直接衰减参数（decoupled）
```

#### Learning Rate Schedule

```
step 0 ~ warmup_iters:    线性增加 lr（从 0 到 learning_rate）
step warmup_iters ~ end:  余弦衰减 lr（从 learning_rate 到 min_lr）

       lr
       ↑
       │    /\
       │   /  \
       │  /    \____
       │ /          \___
       │/               \____
       └──────────────────────→ step
         warmup    cosine decay
```

#### Gradient Clipping

```
如果梯度太大（> max_grad_norm），就等比例缩小
防止一次更新步子太大导致训练崩溃
```

### 动手实验

- [ ] 把 `learning_rate` 从 1e-3 改成 1e-2，观察是否 loss 爆炸
- [ ] 把 `learning_rate` 从 1e-3 改成 1e-5，观察是否训练太慢
- [ ] 去掉 lr decay（改成常数 lr），对比最终 loss
- [ ] 去掉 gradient clipping（设成很大值），在大 lr 下观察是否崩溃
- [ ] 把 `warmup_iters` 从 100 改成 0，观察第一步 loss 是否异常

### 一张"训练稳定性对比图"

记录以下配置的 loss 曲线：

| 配置 | lr | warmup | grad_clip | 结果 |
|------|-----|--------|-----------|------|
| 默认 | 1e-3 | 100 | 1.0 | 正常收敛 |
| 大 lr | 1e-2 | 100 | 1.0 | loss 爆炸 |
| 无 warmup | 1e-3 | 0 | 1.0 | 初期不稳定 |
| 无 lr decay | 1e-3 | 100 | 1.0 | 最终 loss 更高 |

### 输出成果

- 一张"训练稳定性对比图"
- 能解释：warmup、lr decay、gradient clipping 各自的作用

## Day 6：生成机制（Inference）

### 教学目标

理解：**GPT 是怎么"说话"的**

### 核心代码

`model.py` 中的 `generate()` 方法 + `sample.py`

### 生成流程

```
输入 prompt tokens: [t1, t2, t3]
  ↓
model forward → logits for t4
  ↓
logits / temperature
  ↓
top-k 过滤（只保留概率最高的 k 个）
  ↓
softmax → 概率分布
  ↓
按概率随机采样 → t4
  ↓
拼接: [t1, t2, t3, t4]
  ↓
重复，直到生成 EOS 或达到 max_length
```

### 关键概念

#### Temperature

```
temperature = logits / T

T < 1.0: 分布更尖锐 → 更确定 → 更重复
T = 1.0: 原始分布
T > 1.0: 分布更平坦 → 更随机 → 更多样（也更乱）
```

#### Top-k Sampling

```
只保留概率最高的 k 个 token，其余概率设为 0
防止低概率的"垃圾 token"被选中
```

#### Greedy vs Sampling

```
Greedy:  每次选概率最高的 token → 确定性，但容易重复
Sampling: 按概率随机选 → 有变化，但可能选到不好的
```

### 动手实验

- [ ] `--temperature=0.5`：生成保守、重复的文本
- [ ] `--temperature=1.0`：生成平衡的文本
- [ ] `--temperature=1.5`：生成发散、不太通顺的文本
- [ ] `--top_k=1`：等价于 greedy，观察重复现象
- [ ] `--top_k=200`：更自由的采样
- [ ] 对比不同 prompt 的生成效果

### 输出成果

- 三种温度下的生成对比
- 能解释：temperature 和 top-k 各自的作用

## Day 7：改模型（最关键的一天）

### 教学目标

进入"你可以改 GPT 了"的阶段

### 推荐改动（选一个深入做）

#### 选项 A：加 RoPE（旋转位置编码）

**背景**：原始 GPT-2 用绝对位置编码（`wpe`），RoPE 是一种更先进的相对位置编码，被 LLaMA 等现代模型采用。

**改动点**：

1. 在 `CausalSelfAttention` 中加入 RoPE
2. 去掉 `GPT` 中的 `wpe`
3. 在 attention 计算前对 Q、K 应用旋转

**学习收获**：理解位置编码的演进，面试高频题

#### 选项 B：改 RMSNorm

**背景**：LayerNorm 有 bias 项，RMSNorm 去掉了 mean 的计算，只保留 scale，更简单也更快。

**改动点**：

1. 把 `LayerNorm` 替换成 `RMSNorm`
2. 对比训练速度和最终 loss

**学习收获**：理解归一化层的设计空间

#### 选项 C：加 KV Cache

**背景**：推理时每个 token 都要重新算所有 token 的 attention，KV Cache 缓存之前的 K、V 避免重复计算。

**改动点**：

1. 在 `generate()` 中维护一个 KV cache
2. 每次只送入新 token，复用之前的 K、V
3. 对比推理速度

**学习收获**：理解推理优化的核心技术，面试必问

#### 选项 D：在自定义数据上微调

**改动点**：

1. 准备你自己的文本数据（中文、代码、专业领域）
2. 写一个 `prepare.py` 处理数据
3. 用 `init_from='resume'` 或 `init_from='gpt2'` 微调

**学习收获**：端到端理解 fine-tuning 流程
